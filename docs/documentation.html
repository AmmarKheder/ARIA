ARIA Documentation

ARIA (Attention-based Resolution-Integrated Air quality) is a global PM₂.₅ forecasting model at 1km resolution, built on a dual-branch Vision Transformer with cross-attention.

Introduction

ARIA takes ERA5 meteorological reanalysis and CAMS chemistry forecasts as inputs, combines them with static emission proxies (roads, nighttime lights, population), and predicts PM₂.₅ concentration at ~1km resolution anywhere on Earth.

The model operates on 512×512 pixel tiles and can be tiled to cover any region. It predicts a delta on top of the CAMS background: pred = CAMS_t + decoder(tokens), which guarantees physically grounded predictions even at initialization.

ℹ️ ARIA is currently training on 128 MI250X GPUs. Checkpoints will be published on HuggingFace when training completes.

Architecture

ERA5 (72ch, 168×280)  →  Global ViT  (depth=8, dim=768, heads=12)
                              patch_size=8  →  735 tokens
                                                    ↕ Cross-Attention × 2
Local (18ch, 512×512) →  Local ViT   (depth=6, dim=512, heads=8)
                              patch_size=16 →  1024 tokens + skip
                                                    ↓
                                           CNN Decoder
                                           32→64→128→256→512px
                                                    ↓
                                    PM₂.₅ delta @ 1km

Delta prediction

The model predicts a residual on top of CAMS:

pred_pm25 = cams_pm25_interpolated + model(era5, local_input)

The final conv layer is zero-initialized, so the model starts at the CAMS baseline and learns to correct it. This avoids training instability and gives physically meaningful predictions from epoch 0.

Inference

import torch
import numpy as np
from cranpm.training.trainer import CranPMLightning

# Load model
model = CranPMLightning.load_from_checkpoint("aria_global.ckpt")
model.eval().cuda()

# Prepare inputs (example shapes)
era5  = torch.randn(1, 72, 168, 280).cuda().bfloat16()   # global branch
local = torch.randn(1, 18, 512, 512).cuda().bfloat16()   # local branch

with torch.no_grad():
    delta = model.model(era5, local)                      # (1, 1, 512, 512)
    pred  = cams_bilinear + delta.float().cpu().numpy()   # µg/m³
⚠️ Cast bf16 tensors to float32 before .numpy() — bf16 is not supported by NumPy.

Configuration

All training and model parameters live in configs/global_pretrain.yaml:

model:
  era5_channels: 72          # 30+6 ERA5/CAMS at t, 30+6 at t-1
  global_img_size: [168, 280]
  global_patch_size: 8
  global_embed_dim: 768
  global_depth: 8
  local_channels: 18         # CAMS×12 + elev + roads + lights + pop + lat + lon
  local_img_size: [512, 512]
  local_patch_size: 16
  local_embed_dim: 512
  local_depth: 6
  cross_layers: 2

data:
  ghap_mean: 12.0            # normalization: (raw - mean) / std
  ghap_std:  18.0

train:
  batch_size: 2              # per GPU
  learning_rate: 5.0e-5
  epochs: 300
  precision: "bf16-mixed"
  alpha_grad: 0.0           # gradient loss disabled (conflicts with delta prediction)

FAQ

What inputs do I need at inference?

ERA5 (surface + pressure levels) and CAMS for the target date and the previous day. Static layers (elevation, road density, nighttime lights, population) are precomputed once.

Can I run ARIA on CPU?

Yes, but it is slow. For production use, a GPU with at least 16GB VRAM is recommended. The model runs in bf16 by default — use model.float() if your GPU does not support bf16.

How large are the zarr training files?

GHAP global: ~2TB total (5 years). ERA5 global: ~500GB. CAMS: ~200GB. A smaller regional subset is sufficient for fine-tuning.

Can I fine-tune on a specific region?

Yes — pass a regional crop in the config and adjust ghap_h/ghap_w accordingly. The architecture is resolution-agnostic.
