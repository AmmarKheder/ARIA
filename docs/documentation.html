Conditional Flow Matching for Global High-Resolution Air Quality Estimation

1 Conditional Flow Matching

ARIA-Flow uses Conditional Flow Matching (CFM) instead of score-based diffusion. CFM defines a probability path between a base distribution $p_0 = \mathcal{N}(0, I)$ and the target PM2.5 distribution $p_1$ via a simple linear interpolation.

Linear interpolant path:

$$x_t = (1 - t)\, x_0 + t \cdot x_1, \quad t \in [0, 1]$$

where $x_0 \sim \mathcal{N}(0, I)$ is Gaussian noise and $x_1$ is the target PM2.5 field (log1p normalized).

The target velocity field is constant along each path:

$$v^*(x_t, t) = x_1 - x_0$$

The model learns $v_\theta(x_t, t, \text{cond}) \approx x_1 - x_0$ with MSE loss:

$$\mathcal{L}_{\text{FM}} = \mathbb{E}_{t, x_0, x_1} \left[ \| v_\theta(x_t, t, \text{cond}) - (x_1 - x_0) \|^2 \right]$$

At inference, we solve the ODE from $x_0 \sim \mathcal{N}(0, I)$ using $N=20$ Euler steps:

$$x_{t+\Delta t} = x_t + \Delta t \cdot v_\theta(x_t, t, \text{cond}), \quad \Delta t = \frac{1}{N}$$

Why Flow Matching over Diffusion?

  • ✓ Straight ODE paths = only 20 steps vs 100+ for DDPM
  • ✓ No noise schedule to tune (vs $\beta_t$ in diffusion)
  • ✓ Stable training — MSE on velocity, no variance explosion
  • ✓ Natural ensemble sampling: run N independent ODE trajectories for uncertainty

2 Model Architecture (107.7M Parameters)

2.1 AdaLN-Zero Time Conditioning

The flow time $t \in [0,1]$ is embedded via sinusoidal positional encoding (256 frequencies) and projected through an MLP. Each transformer block in the local branch uses Adaptive Layer Normalization with zero initialization (AdaLN-Zero) for conditioning:

Time embedding:

$$c = \text{MLP}\left(\left[\sin(t \cdot 1000 \cdot \omega_k),\; \cos(t \cdot 1000 \cdot \omega_k),\; \tau_{\text{lead}}\right]\right) \in \mathbb{R}^{512}$$

AdaLN-Zero modulation (6 parameters per block):

$$(\gamma_1, \beta_1, \alpha_1, \gamma_2, \beta_2, \alpha_2) = \text{SiLU}(\text{Linear}(c))$$ $$h = x + \alpha_1 \cdot \text{Attn}\left(\text{LN}(x) \cdot (1 + \gamma_1) + \beta_1\right)$$ $$\text{out} = h + \alpha_2 \cdot \text{MLP}\left(\text{LN}(h) \cdot (1 + \gamma_2) + \beta_2\right)$$

All modulation weights initialized to zero → $\alpha_i = 0$ at init → each block is an identity function at initialization.

2.2 Topology-Aware Attention (Block 0)

The first block of the local branch uses elevation-informed attention with relative position bias:

$$\text{Attn}(Q, K, V) = \text{softmax}\left(\frac{Q \cdot K^T}{\sqrt{d}} + B_{\text{elev}} + B_{\text{pos}}\right) V$$

Pre-scaling: $(Q \cdot \text{scale}) \times K^T$ to prevent bf16 overflow in the matmul.

2.3 Patch Embedding (MIOpen Bug Workaround)

Due to a verified MIOpen Conv2d bug on AMD MI250X GPUs that produces corrupted outputs (~1e27 values), patch embedding uses F.unfold + nn.Linear instead of nn.Conv2d: 

# Instead of nn.Conv2d(19, 512, kernel_size=16, stride=16)
patches = F.unfold(x, kernel_size=16, stride=16)  # (B, 19*16*16, 1024)
tokens = self.proj(patches.transpose(1, 2))        # (B, 1024, 512)

3 Data Processing

3.1 Target Normalization (GHAP PM2.5)

Log1p transform with fixed global statistics:

$$\hat{y} = \frac{\log(1 + y) - \mu}{\sigma}, \quad \mu = 2.416, \; \sigma = 0.800$$

Land mask threshold: $\hat{y} > -\mu/\sigma = -3.02$ (i.e., raw PM2.5 > 0).

Critical: using $\hat{y} > 0$ would exclude clean air pixels with raw PM2.5 < 15 μg/m³.

3.2 Road Density Normalization

Each OSM road segment contributes a weight to the raster grid based on highway type:

Highway Type Weight Highway Type Weight
motorway8.0secondary3.0
trunk6.0tertiary2.0
primary5.0residential0.8

Normalization to [0, 1] via log1p with 99th percentile clipping:

$$\text{road}_{\text{norm}}(i, j) = \text{clip}\left(\frac{\log(1 + \text{road}_{\text{raw}}(i, j))}{\log(1 + P_{99})},\; 0,\; 1\right)$$

where $P_{99}$ is the 99th percentile of $\log(1 + \text{road}_{\text{raw}})$ over all non-zero land pixels.

3.3 Counterfactual Road Modification

To simulate a highway traffic reduction of $\alpha \in [-1, 1]$:

$$\text{road}_{\text{new}} = \text{clip}\left(\text{road}_{\text{total}} - \alpha \cdot \text{road}_{\text{highway}},\; 0,\; \infty\right)$$ $$\text{tensor}_{\text{new}} = \text{clip}\left(\frac{\log(1 + \text{road}_{\text{new}})}{\log(1 + P_{99})},\; 0,\; 1\right)$$

This replaces channel 13 (road density) in the 19-channel input tensor before running the flow ODE.

3.4 Loss Function

Land-weighted flow matching loss with ocean penalty:

$$\mathcal{L} = \frac{\sum_{i \in \text{land}} \|v_\theta - v^*\|_i^2}{|\text{land}|} + 0.1 \cdot \frac{\sum_{i \in \text{ocean}} \|v_\theta - v^*\|_i^2}{|\text{ocean}|}$$

Ocean weight = 0.1 prevents coastline artifacts. Denominator clamped to min=1.0 to avoid DDP deadlock from disconnected gradients.

Known Hardware Issues (AMD MI250X / ROCm)

BUG MIOpen Conv2d: Produces ~1e27 values for patch_embed (kernel=8, stride=8). Fix: F.unfold + nn.Linear.
BUG bf16 overflow: Q@K^T overflows before scaling. Fix: pre-scale queries (Q*scale)@K^T.
BUG DDP deadlock: Masked loss returning disconnected tensor. Fix: clamp denominator to min=1.0.
TIP MIOpen cache: /tmp is node-local on LUMI. Use srun --ntasks-per-node=1 to create dirs on ALL nodes.
